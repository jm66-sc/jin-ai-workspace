# SOP-共创学习体系 v2（正式版）

> Boss 口述确认 | Claude 整理 | 2026-04-08

---

## 核心理念

> AI 搜集 → 出卡片 → Boss + Claude 双认知批注 → 金龙入库 iMA → 场景化复用落地

只有"原文 + 中心思想 + Boss认知 + Claude认知"四层俱全的完整卡片，才有入库资格。

---

## 完整流程（5步闭环）

### 第1步：信息采集（麦龙 + 金龙）

- **麦龙**：国内源（B站/公众号/博客/知乎/GitHub Trending），每天约 10 条
- **金龙**：海外源（YouTube/Twitter），代理恢复后接入
- 原文（或视频转文字稿）存入 GitHub `共创资料池/原文存档/YYYY-MM/`

---

### 第2步：生成中心思想卡片（麦龙）

每篇原文自动生成极简摘要卡片，存入 GitHub `共创资料池/中心思想卡片/YYYY-MM/`

**卡片格式：**

```
# 卡片-序号-来源-标题一句话核心

**来源**：XX平台 | **日期**：YYYY-MM-DD
**中心思想**：（1-2句话，精准提炼）

**对我们工作的启示**：（麦龙初判，1-3条）

---
【Boss认知】← 留空，等Boss批注
【Claude认知】← 留空，等Claude批注
```

---

### 第3步：双认知批注（Boss + Claude，异步进行）

Boss 随时说"看今天的卡片"或"给我推一篇"：

**Boss这边：**
- 麦龙推出卡片，Boss 口头说出理解、判断、点评
- 麦龙将 Boss 原话/感受记录填入【Boss认知】栏

**Claude这边：**
- 同步读同一张卡片
- Claude 写出自己的认知、解读、延伸思考，填入【Claude认知】栏

> **触发方式**：Boss 对Claude说"这张卡片你来看一下" 或 "一起来看今天的卡片"

---

### 第4步：金龙入库 iMA（金龙）

只有【Boss认知】和【Claude认知】**都填写完毕**的卡片，才由金龙推送到 iMA 对应文件夹。

**iMA 文件夹结构（金龙来建）：**
```
AI学习知识库/
├── 实操技巧/          ← 实操教程类卡片
├── 开源项目/          ← GitHub工具/框架类卡片
├── 行业趋势/          ← 动态/分析类卡片
└── 落地讨论/          ← 已讨论如何在工作中应用的记录
```

---

### 第5步：场景化复用（随时触发）

要讨论落地方案时：
- 直接从 iMA 调取相关卡片（所有人都读过，有共同语言）
- 基于"原文 + 双重认知"一起讨论
- 形成共识 → 写入 TASKS.md → 执行

---

## 每日推送内容比例（10条）

| 类别 | 条数 | 说明 |
|------|------|------|
| 实操教程 | 2-3 | 有具体步骤，可马上跟着做 |
| GitHub新项目 | 2-3 | 与我们工作相关的新开源工具 |
| 优质内容 | 2-3 | 有洞见的文章/案例/深度分析 |
| 趋势动态 | 1-2 | 行业大事/新模型/框架更新 |

---

## 角色分工

| 角色 | 负责 |
|------|------|
| **麦龙** | 国内源采集 + 原文存档 + 生成卡片 + 推送 + 记录Boss认知 |
| **金龙** | 海外源采集（代理恢复后）+ iMA 建库 + 推送完整卡片入库 |
| **Claude** | 读卡片 + 写Claude认知 + 讨论时协助落地分析 |
| **Boss** | 阅读卡片 + 口头批注 + 最终决策哪些落地 |
