# 协作规则（协议）

> 所有代理必须遵守本规则

---

## 1. 指令层级

```
Boss（最高决策）
    ↓
Claude（战略层 / 总参谋）
    ↓           ↓
  金龙         麦龙
（本地执行）  （云端执行）
```

- Boss → Claude：终端对话 / 直接指令
- Boss → 金龙：桌面交互 / 紧急时微信远程
- Boss → 麦龙：微信发送指令
- Claude → 金龙 / 麦龙：写入 TASKS.md，对方轮询执行

---

## 2. 任务执行规范

**领取任务：**
1. 在 TASKS.md "待执行" 中找到自己标注的任务
2. 将任务移至"执行中"，注明时间

**完成任务：**
1. 将结果存入指定输出路径
2. 将任务移至"已完成"，注明完成时间
3. 更新 STATUS.md 中自己的状态

**遇到问题：**
- 在 TASKS.md 任务行后加注 ⚠️ + 问题描述
- 等待 Colode 或 Boss 处理

---

## 3. 内容流水线规则

```
选题（Boss / Colode）
    ↓
爱马内部生成大纲 + 初稿框架
    ↓
金龙：补充干货、案例、数据
    → 存入 content/drafts/YYYYMMDD_标题.md
    ↓
Claude：审核，提修改意见
    → 意见写在文件头部 [REVIEW] 块中
    ↓
金龙：二改
    → 存入 content/review/YYYYMMDD_标题_v2.md
    ↓
Boss 确认 → 发布
    → 存入 content/published/
```

---

## 4. 文件命名规范

- 内容文件：`YYYYMMDD_标题关键词.md`（如 `20260401_AI报价工具实战.md`）
- 情报文件：`YYYYMMDD_来源_摘要.md`（如 `20260401_工控自动化_3篇摘要.md`）
- 知识文件：按品牌/类别建子目录（如 `knowledge/siemens/s7-1200-手册.pdf`）

---

## 5. 禁止事项

- 不得修改 Boss 明确批准的最终方案
- 不得在公开仓库中上传含客户信息的文件
- 不得删除"已完成"记录（保留历史）
