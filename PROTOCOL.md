# PROTOCOL.md — 团队协作协议

> 版本：v2
> 更新：2026-04-06
> 更新者：麦龙

---

## 核心原则：所有工作产出必须同步 GitHub

**麦龙（云端 Agent）做任何事情后，必须将结果写入 GitHub 仓库，让金龙等团队成员能够同步读取并继续执行。**

---

## 麦龙工作同步规则

### 必须同步的情形

- 调研报告 / 分析结论
- 方案 / 提案 / 建议
- 工作流程设计
- 工具盘点结果
- 监控 / 爬取结果
- 沙盘推演记录
- 决策逻辑说明
- Boss 交代的任务完成报告

### 同步位置约定

| 内容类型 | 写入目录 |
|---------|---------|
| 分析 / 调研报告 | `briefs/` |
| 方案 / 提案 | `briefs/` 或 `protocols/` |
| 工具盘点 | `tools/inventory-{角色}.md` |
| 监控结果 / 爬取报告 | `intel/` |
| 每日工作记录 | `memory/` |
| 沙盘推演 | `briefs/simulations/` |
| 决策记录 | `briefs/decisions/` |

### 同步时机

**每完成一项独立任务，立即同步 GitHub，无需等待。**

---

## TASKS.md 同步规则

- 麦龙、金龙各自定时（每日 09:00 / 21:00）扫描 TASKS.md
- 完成后将任务移至"已完成"区，注明完成时间和结果
- 有新任务或临时任务，直接写入 TASKS.md 对应区域

---

## 交接机制

```
Boss 发指令 → 任意一方领取 → 执行 → 写入 GitHub → 推送微信通知
```

所有任务以 GitHub 为唯一事实来源，各 Agent 独立扫描、独立执行。

---

## 例行工作（麦龙）

- 每日 09:00 / 21:00：git pull → 扫描 TASKS.md → 执行属于麦龙的任务
- 任何任务完成后：结果同步 GitHub + 微信通知 Boss
- 任何任务完成后：更新 TASKS.md 对应任务状态
