## 卡片①（⭐⭐⭐最高）Actor模型：多Agent协作的底层原理

**来源**：博客园·Serverless（2026-03-17）| 原文已存档

### 原理（详细解释）

**什么是 Actor 模型？**

OpenClaw 的多 Agent 架构来自计算机科学的 Actor Model 理论：

```
每个 Agent = 一个独立的"Actor"（演员）
- 独立的工作内存（记忆）
- 独立的文件系统（只属于自己的文件）
- 独立的身份定义（SOUL.md）
- 只能通过"发消息"与其他 Agent 通信
- 不能直接读其他 Agent 的内存
```

**为什么这样设计？**

传统程序：所有函数共享同一块内存，一个出错全部崩溃。
Actor 模型：每个 Actor 完全独立，一个崩溃不影响其他，各自处理各自的消息队列。

**Gateway 的角色：**

```
用户 → 发消息给 Gateway
       ↓
Gateway = 邮递员/交换机
       ↓
精准分发到目标 Agent
```

### 方法（5条可操作步骤）

1. **配置 SOUL.md**：每个 Agent 的身份配置，写清楚"我是谁、我的职责、我的专长"
2. **配置 AGENTS.md**：团队通讯录，写清楚"团队里有哪些 Agent、它们各自负责什么"
3. **建立共享文件区**：`/data/.openclaw/shared/` 作为团队协作的共同工作台
4. **CLI 一键组队**：用命令行描述团队结构，自动完成所有 Agent 的配置
5. **汇报协议**：每个 Agent 的输出必须写入 AGENTS.md，形成可追溯的协作记录

### 效果（可量化的结果）

| 效果维度 | 描述 |
|---------|------|
| 组织性 | 从"单兵作战" → 有组织的团队协作 |
| 可追溯性 | 每一步操作都有记录，用户可查看全流程 |
| 人工介入 | 从全程参与 → 仅监督和调度 |
| 扩展性 | 加新 Agent 只需配置 SOUL.md，不影响现有 Agent |

### 对我们的直接价值

麦龙 + 金龙 + Cloud Code 的架构，正好对应这个模型：
- **麦龙**：调度层 = Gateway（接收任务、分发给金龙/Cloud Code）
- **金龙**：执行层 = 专业 Agent（负责爬虫、浏览器操作）
- **Cloud Code**：辅助层 = 辅助 Agent（负责编程任务）

## 卡片②（⭐⭐第二）5个AI员工：从需求到代码自动流转

**来源**：腾讯云开发者社区（2026-04）| 原文已存档

### 原理（详细解释）

**三层架构的工作原理：**

```
第一层：交互层（用户界面）
  ↓ 用户下达任务
第二层：调度层（任务分解）
  ↓ 拆解成子任务
第三层：执行层（专业Agent）
  ↓ 各自执行
汇总输出 → 用户
```

**为什么能这么快？**

核心：专业化。每个 Agent 只做一件事。
- 不是让一个很强的 AI 做所有事
- 是让很多个专用的 AI 同时做各自的事

类比：医院的分工
- 不是让一个医生看所有科
- 是专科医生各司其职，疑难杂症才上会诊

### 方法（5条可操作步骤）

1. **定义角色**：明确每个 Agent 的专长（爬虫/解析/通知/存档）
2. **建立流程**：任务进入 → 调度层分解 → 分发给专业 Agent → 汇总输出
3. **自动触发**：上游输出完成后自动触发下游，无需人工干预
4. **结果记录**：每个 Agent 的输出写入共享文件区
5. **人工兜底**：所有 Agent 输出由调度层汇总后交人工确认

### 效果（可量化的结果）

| 指标 | 传统方式 | 多Agent协作 |
|------|---------|-----------|
| 耗时 | 1-2天 | **15-30分钟** |
| 人工介入 | 全程参与 | 仅监督 |
| 出错率 | 高（人工疲劳） | 低（Agent专注单一任务） |
| 可追溯性 | 低（口头传递） | 高（每步有记录） |

### 对我们的直接价值

```
招标信息处理流程：
① 麦龙（爬虫）→ 收集信息
② 金龙（解析）→ 判断是否入库
③ 麦龙（汇总）→ 提取关键字段
④ 钉钉通知 → 推送给Boss
```

这条流水线是真实的，不是想象。

## 卡片③（⭐高）Skill一键组建团队

**来源**：博客园·Serverless（2026-03-17）| 原文已存档

### 原理（详细解释）

**Skill 在 OpenClaw 里是什么？**

Skill = 给 Agent 安装的"技能包"
- 包含专项知识
- 包含操作规程
- 用 SKILL.md 文件描述
- 安装后 Agent 就具备了这个专项能力

**为什么 Skill 能快速组建团队？**

传统方式：手动配置每个 Agent 的 SOUL.md、通讯录、文件权限
Skill 方式：说一句"我要组建一个爬虫团队"，系统自动配置好所有东西

类比：传统 = 手动组装电脑；Skill = 购买整机，插电就用

### 方法（3条核心步骤）

1. **CLI 一键配置**：运行 `openclaw agents add --team` 描述团队角色，自动完成身份配置
2. **钉钉绑定**：配置钉钉机器人，在钉钉群里 @Agent 名称直接下达任务
3. **统一凭证**：所有 Agent 共用同一凭证接入钉钉，简化管理

### 效果

- **组建时间**：从几小时 → 几分钟
- **使用门槛**：不需要懂技术，描述清楚就行
- **调度方式**：钉钉群里 @名字 = 指挥 Agent

### 对我们的直接价值

Boss 在钉钉里直接 @麦龙 或 @金龙 = 指挥多虾团队
不需要打开 OpenClaw，钉钉就是入口
