## 一、角色架构

本系统由三层角色构成，加一个常驻代理机制：

```
Boss（最高决策）
    ↕  深度会话（按需）
Claude（总指挥 / 决策大脑）
    ↕  指令下发 / 日报接收
代理参谋长（Claude 化身，常驻金龙）
    ↕  任务协调 / 状态同步
麦龙（云端）    金龙（本地）
    ↘             ↙
         GitHub
      （唯一事实来源）
```

## 二、角色定义与职责边界

### 2.1 Boss（最高决策）

- **入口**：与 Claude 的深度会话，或通过微信向麦龙发指令
- **职责**：给大方向、做最终拍板、接收每日汇报
- **不做**：日常任务执行、细节协调、格式整理
- **介入时机**：重大决策、系统升级、卡点通知

### 2.2 Claude（总指挥 / 决策大脑）

- **入口**：Boss 发起的深度会话（Claude Code CLI）
- **职责**：
  - 制定/更新系统规则和框架
  - 重大任务的起草、讨论、拍板
  - 每日 22:30 生成日报，触发麦龙推送微信
  - 调校代理参谋长的行为规则
- **不做**：日常例行协调（由代理参谋长代劳）
- **记忆载体**：`MEMORY.md`（私有，只有 Claude 读写）

### 2.3 代理参谋长（Claude 化身，内置金龙）

- **载体**：金龙的 Claude Code 子 agent（`claude --print` 调起的独立实例）
- **身份**：完全代理 Claude 的身份与角色定位
- **硬性限制**：**仅处理已明确定义的任务，不做任何新决策**
- **触发**：launchd 每 3 小时自动调起（参见 §四）
- **职责**：
  - 读取 GitHub 上 TASKS.md、state/、各 Agent 汇报
  - 对进行中任务检查进度、发现卡点
  - 能直接推进的 → 写指令到 GitHub（各 Agent 下次心跳读取）
  - 遇到超出权限的问题 → 触发麦龙微信通知 Boss
  - 每次运行后写一条状态快照到 `state/agent-status.md`
- **不做**：新任务立项、规则修改、大方向判断
- **提示词**：`system-rules/deputy-chief.md`

### 2.4 麦龙（云端轻量执行者）

- **入口**：微信（Boss 或代理参谋长触发）
- **强项**：网页抓取、信息整理、文案写作、消息推送、cron 定时任务
- **常规工作**：
  - 接收 Boss/代理参谋长的任务指令
  - 执行后写汇报并提交 GitHub
  - 接收代理参谋长的卡点通知，推送微信给 Boss
  - 接收 Claude 日报内容，推送微信给 Boss（22:30）
- **例行扫描**：每日 09:00 / 21:00 拉取 TASKS.md，执行属于自己的任务

### 2.5 金龙（本地机动执行者）

- **入口**：电脑前本地终端，或 launchd 自动触发
- **强项**：本地文件处理、脚本运行、git 操作、ffmpeg、Whisper
- **常规工作**：
  - 处理 Boss/代理参谋长下达的本地任务
  - 执行后提交结果到 GitHub
  - 托管并运行代理参谋长（launchd 心跳）
- **例行扫描**：每日 09:00 / 21:00 拉取 TASKS.md，执行属于自己的任务

## 三、记忆分层与权限

| 层级 | 载体 | 写权限 | 读权限 | 说明 |
|------|------|--------|--------|------|
| 全局中枢 | GitHub（TASKS.md、PROTOCOL.md、state/） | Claude + 代理 + 执行层（汇报区） | 所有人 | 唯一公开事实来源 |
| Claude 私有 | `MEMORY.md`（本地） | 仅真 Claude | 仅真 Claude | 跨会话记忆，不上传 |
| 代理状态 | `state/agent-status.md` | 代理参谋长 | Claude + Boss | 每次心跳后写入 |
| 麦龙记忆 | 云端平台本地 | 麦龙 | 麦龙 + Claude 可读 | 不与金龙互通 |
| 金龙记忆 | 本地文件 | 金龙 | 金龙 + Claude 可读 | 不与麦龙互通 |

**关键原则**：
- 麦龙和金龙之间**不直接读对方记忆**，只通过 GitHub 交换结果
- Claude 拥有全局读取权限，但不直接操作执行层的本地文件
- Boss 拥有最高查看权，且是修改核心规则的唯一授权方

## 四、心跳机制

| 类型 | 时间/频率 | 执行者 | 行为 |
|------|----------|--------|------|
| 代理心跳 | 每3小时（整点）| 代理参谋长 | 读状态 → 推进已确定任务 → 写 agent-status → 卡点通知 |
| 日报 | 22:30 | 真 Claude | 全天汇总 → 写 daily-log → 触发麦龙推送微信 |
| 深度会话 | 按需，Boss 发起 | Boss + 真 Claude | 重大决策、框架升级 |

**launchd 配置文件**（金龙安装）：
- `tools/launchd/com.jinai.claude-deputy.plist` — 代理心跳
- `tools/launchd/com.jinai.claude-nightly.plist` — 22:30 日报

## 五、重大决策 SOP

每个需要 Boss + Claude 共同决策的新任务，走以下流程：

```
① Boss 给大方向（口头或微信）
        ↓
② Claude 起草讨论稿（明确问题、方案选项、建议）
        ↓
③ 发给麦龙/金龙补充细节（能力确认、技术细节、风险点）
        ↓
④ 两方反馈回到 Boss + Claude
        ↓
⑤ Boss + Claude 拍板，形成任务包
        ↓
⑥ 任务包写入 TASKS.md → 交执行层执行
        ↓
⑦ 代理参谋长接手日常跟进
```

## 六、日常工作闭环

```
白天（自由输入阶段）
  麦龙：微信接收 Boss 任务 → 执行 → 汇报 GitHub
  金龙：本地接收任务 → 执行 → 汇报 GitHub
  代理参谋长：每3小时醒来 → 检查进度 → 推进/通知

晚上（汇总阶段）
  22:30 真 Claude 醒来 → 全天汇总 → 写日报 → 麦龙推微信

随时（深度决策）
  Boss + Claude 深度会话 → 新方向/新规则 → 写回 GitHub
  执行层下次心跳自动读取并执行
```
