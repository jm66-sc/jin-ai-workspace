# T025 任务说明：技能扩展调研·武装云端小龙虾

> 发布时间：2026-04-08 | 发布者：Claude（经Boss指令）
> 执行者：麦龙

## 任务背景

Boss 观察认为：麦龙作为云端小龙虾，当前能力虽然够用但**不够武装**。需要扩展技能（"scale"），打造一个真正**24小时不睡觉、不受环境限制**的云端待命Agent。

## 调研目标

1. **摸清技能生态**：Claude Hub（或类似技能市场）上有哪些可用技能
2. **评估适配性**：哪些技能适合麦龙当前平台（MaxClaw/OpenClaw）
3. **制定武装方案**：由 Boss + Claude 共同决策安装哪些技能

## 核心问题

### 1. "Claude Hub"到底是什么？
- 是 Anthropic Claude 的官方技能市场？
- 是 Claude Code 的插件库？
- 还是 MaxClaw/OpenClaw 平台的技能商店？
- **第一步：搞清楚这个"Hub"的具体位置和访问方式**

### 2. 麦龙需要哪些能力扩展？
基于当前工作需求，优先级排序：
1. **定时任务增强**：更稳定的cron调度、任务队列管理
2. **监控能力**：多源信息监控、异常检测
3. **数据处理**：结构化数据提取、批量处理
4. **通信扩展**：更多消息通道支持（邮件、Slack等）
5. **工具集成**：API调用、第三方服务集成

### 3. 安装可行性评估
- 技能安装是否需要管理员权限？
- 是否需要代码部署？
- 是否有Token消耗？
- 是否与现有工具链兼容？

## 调研方法建议

1. **先定位**：搞清楚"Claude Hub"的具体网址/访问方式
2. **分类扫**：按功能分类浏览技能，记录关键信息
3. **深度评**：对高潜力技能进行深度评估（阅读文档、安装说明）
4. **横向比**：对比类似技能，选择最优

## 时间要求

- **调研时间**：2-3天（细致调研，不匆忙）
- **报告质量**：深度优先于速度，宁缺毋滥
- **决策方式**：报告完成后，Boss + Claude 共同审议决定安装清单

## 特别提醒

**不要假设**麦龙能直接安装所有技能。有些可能需要：
- 金龙本地辅助安装
- Boss 提供 API Key
- 平台管理员权限

**实事求是**：能装的如实记录，不能装的明确说明原因。
