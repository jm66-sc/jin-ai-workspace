## 第三步：视觉导航 — 找到列表页 URL（核心任务）

**工具**：使用你集成的港大视觉智能能力（GUI/视觉模拟操作），模拟人类打开浏览器、点击按钮，找到两个目标网站的招标公告列表页 URL。

### 目标一：中国政府采购网（ccgp.gov.cn）

**入口**：https://www.ccgp.gov.cn/

操作步骤：
1. 打开首页
2. 找到"采购公告"或"招标公告"入口（不需要登录）
3. 找到可以按关键词搜索的列表页
4. 目标 URL 格式应类似：`https://search.ccgp.gov.cn/bxsearch?searchtype=2&page_index=1&kw=消防设备`
5. 验证：用关键词"消防设备"搜索，确认能看到招标公告列表

**注意**：ccgp.gov.cn 有 IP 频率限制，请求间隔不少于 10 秒。如被封，停止，在报告中注明。

### 目标二：成都建工集团 或 成都华西集团

**操作步骤**：
1. 用搜索引擎（或直接访问官网）找到成都建工集团或成都华西集团的官方招标采购入口
2. 判断是否需要注册：
   - **无需注册或注册无门槛**：自行完成，记录注册用的信息
   - **需要人工辅助（扫码/验证码/复杂资料）**：先停下，在报告里注明"需要 Boss 配合注册"，把注册页面截图或URL写进报告
3. 找到招标公告列表页 URL（包含消防/机电/楼控相关公告的那个入口）

**结果**：把找到的列表页 URL 写入 `briefs/t013-report.md`

## 第四步：列表页爬取

使用以下配置（已验证有效，不要改动核心参数）：

```python
from crawl4ai import AsyncWebCrawler, BrowserConfig

browser_config = BrowserConfig(
    browser_mode="undetected",  # 核心！不能改
    enable_stealth=True,         # 核心！不能改
    headless=True
)
```

对每个找到的列表页 URL，爬取前 2-3 页，提取：
- `title`：公告标题
- `detail_url`：公告详情页 URL
- `source`：来源网站

输出格式（JSON）：
```json
[
  {"title": "某某单位消防设备采购公告", "detail_url": "https://...", "source": "ccgp"},
  ...
]
```

**输出文件**：
- `intel/raw-list-ccgp.json`（政府采购网结果）
- `intel/raw-list-成都建工.json`（成都建工/华西结果）

每个文件至少包含 20 条以上原始条目（筛选前）。
