## 一、背景与战略意图

Boss 有一套已有的政府采购招标爬虫工具（SmartScout），原本设计给**人**使用，有界面、有操作流程。

**现在要做一件根本性的转变：**

> 把这套工具重构为 **Agent 原生工作流**。
> 未来不是人去点鼠标爬数据，而是人只说一句话（比如"帮我监控西门子的招标公告"），
> Agent 自己完成全部工作：找URL、爬列表、筛选、读详情、输出结果、推送通知。

这不是修一个 bug，是定义一种新的工作方式。

## 二、现有资产（请认真阅读）

代码在：`tools/crawler-金龙/SmartScout/`

**要保留的核心资产：**
1. **Chrome undetected 模式**：`browser_mode="undetected"` + `enable_stealth=True` — 这是突破政府网站反爬的关键配置，已验证有效
2. **两层漏斗逻辑**：列表页先批量抓标题 → 粗筛 → 只打开有价值的详情页（节省资源）
3. **SQLite 规则库**：积累的筛选规则，有长期价值

**要废弃的：**
- Streamlit 界面（Agent 不需要）
- FastAPI 服务层（Agent 直接调脚本，不需要 HTTP）
- 有 bug 的生产者-消费者流水线（需修复或重写）

## 三、目标工作流（初稿，待你们修正）

```
用户输入：单位名称（如"西门子"）
    ↓
[Step 0] URL发现：根据单位名找到政府采购列表页URL
    ↓
[Step 1] 列表页爬取：抓取该页所有条目（标题 + 详情URL）
    ↓
[Step 2] 第一层筛选：AI快速判断标题相关性，过滤无关条目
    ↓
[Step 3] 详情页读取：打开筛选后的URL，提取正文内容
    ↓
[Step 4] 结构化输出：写入 intel/ 目录 + 微信通知 Boss
```

每一步的输出写入 GitHub 固定文件，下一步 Agent 读取后接力执行。
