## 二、需要 Boss 决策的三个关键点

### 决策点 A：知识库放哪？

麦龙报告假设用 GitHub 文件夹结构，但 MEMORY.md 里记着这个问题还没讨论定。

| 方案 | 优点 | 缺点 |
|------|------|------|
| **GitHub JSON**（麦龙方案） | 结构化，麦龙/金龙都能读写，版本追踪 | Boss 看内容需要打开文件，不直观 |
| **IMA 知识库** | Boss 自己看最顺手，支持搜索 | 封闭，麦龙写入需要 API 权限 |
| **两层结合**（推荐） | GitHub 存原始归档，IMA 存"强烈推荐亲读"精选 | 维护两套，稍麻烦 |

**建议：先确认知识库选型，再建文件夹结构，否则白建。**

---

### 决策点 B：每日推送的量和质如何把控？

报告说每天 10 条，每日 09:00 推送。实际执行有两个问题：

**问题 1：麦龙有速率限制**
- 麦龙每次只能跑小批量（T018 也是 10 篇/次）
- 每日采集 + 判断价值 + 推送，是三步独立操作，要确认麦龙能不能稳定每天做

**问题 2：筛选标准没有**
- 什么是"值得推送"的内容？现在没有阈值
- 建议定：只推送满足以下至少一条的内容
  - 与当前工作（招标/内容流水线/Agent 协作）有直接关联
  - 有具体操作步骤（tutorial/实战）
  - 开源项目 Stars 增量 > 500/天 或 总量 > 5000

**可以先跑"样刊"验证**：让麦龙先出 1-3 期样刊，Boss 看完再定正式标准，比直接拍板更稳。

---

### 决策点 C：金龙负责的海外源，现实吗？

报告把 Twitter/X 监控分给金龙，但：
- Twitter 需要注册账号（目前没有）
- YouTube 监控没有具体工具
- GitHub Trending 报告里同时列为"B类需代理"又分给了麦龙，**存在矛盾**

**建议澄清：**
- GitHub Trending：麦龙直接访问（不需代理），归麦龙
- Anthropic/OpenAI 官方博客：金龙（需代理，但内容更新不定期）
- Twitter：暂缓，等金龙账号问题解决后再说
- YouTube：暂缓，先聚焦文字内容

## 三、整体方案建议（讨论版）

### 第一阶段：验证期（2周）

**目标：跑通流程，找到质量基准**

1. 麦龙：每日从掘金 + 博客园 + GitHub Trending 采集，整理 10 条推送，微信发给 Boss
2. Boss：每天花 5 分钟标注"有用 / 没用 / 还行"，反馈给麦龙
3. 不建知识库，先把推送质量跑稳

**麦龙承担的工作量评估：**
- 每日采集：约 2-3 次 batch_web_search
- 价值判断：LLM 判断，消耗 token 可控
- 推送：1次 message 发微信
- 结论：**麦龙有能力承担，但要避免跟 T018 冲突，建议分开时间段跑**

### 第二阶段：建库期（第3-4周，视第一阶段效果）

- 根据 Boss 的标注反馈，定出筛选标准
- 按确定的知识库方案开始归档
- 金龙的海外源在代理稳定后接入
