## 第三部分：任务能力边界

**10. 定时拉取 URL 列表并爬取**
- ✅ 可以
- 方式：cron 定时任务 + HTTP 请求批量爬取
- 支持：robots.txt 合规爬取、页面内容提取

**11. 调用大模型进行语义解析/摘要**
- ✅ 可以
- 集成：MiniMax / DeepSeek 模型直接调用
- 能力：摘要提取、关键词抽取、情感分析、主题分类

**12. 将处理结果写回 GitHub**
- ✅ 可以
- 流程：生成文件 → git add/commit/push
- 刚完成操作：克隆仓库并写入文件

**13. 定时任务能力**
- 最小间隔：1 秒（通过 cron 表达式控制）
- 并发数量：可多个定时任务同时运行
- 稳定性：✅ 稳定，容器重启后自动恢复
- 限制：部分平台级限制需遵守

**14. 最擅长的 3 件事**
1. **信息检索与调研分析**：给定主题，系统性搜集、整理、归纳信息
2. **文案撰写与结构化输出**：报告、方案、邮件、纪要等各类文字输出
3. **跨平台信息整合**：将来自不同来源（微信/钉钉/GitHub/网页）的碎片信息汇聚为完整视图

**15. 做不到或很吃力的 3 件事**
1. **需要本地物理交互的操作**：如必须操作本地电脑鼠标/键盘的场景（需要金龙配合）
2. **需要图形界面辅助的调试**：纯终端难以完成的复杂前端调试
3. **需要持续占用本地资源的任务**：长时间跑重计算任务（受限于容器资源）
