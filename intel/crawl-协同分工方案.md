## 核心协同原则

| 环节 | 谁来做 | 原因 |
|------|--------|------|
| 需要浏览器操作（登录/注册/验证码） | **金龙本地** | 有眼睛看、有手点 |
| 规模化数据爬取（定时/大批量） | **麦龙云端** | 定时稳定，不占本地资源 |
| 信息汇总 + 分析 + 推送 | **麦龙云端** | 大模型理解能力强 |

---

## 协同流程

```
第一步：注册账号
    ↓
金龙本地：打开网站 → 填写注册信息 → 通过验证码 → 完成注册
    ↓
金龙本地：登录 → 提取 cookie 或 API token
    ↓
金龙：将凭证写入 GitHub：intel/credentials/{平台名}.md（仅凭证，勿写明文密码）
    ↓
麦龙：读取凭证 → 定时爬取 + 分析 → 推送通知
```

## 凭证存储格式（GitHub）

```
intel/
└── credentials/
    ├── ec-powerchina-cn.md      # 格式：username + token（不含密码）
    └── chinabidding-com-cn.md
```

**格式约定：**
- 文件名：{平台域名}.md
- 内容：username + token/API key
- ⚠️ 不写入明文密码，只写 username + token
- ⚠️ 凭证文件加入 .gitignore，不提交到公开仓库（建议建私有仓）
