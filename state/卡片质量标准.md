## 一、核心问题：昨天发生了什么？

### 不合格案例（昨天的情况）
```markdown
### 方法（提炼）
- [ ] 方法1：多 Agent 接力工作流设计
- [ ] 方法2：每个 Agent 专注单一任务
- [ ] 方法3：输出自动流转到下一个 Agent
```
**问题**：只有标题框架，没有具体内容。属于"空壳卡片"。

### 合格标准（今天起必须做到）
```markdown
### 方法（提炼）
- [ ] **方法1**：设计多Agent接力工作流时，采用"任务队列+状态机"模式。具体步骤：① 定义每个Agent的输入输出接口 ② 建立任务队列和状态追踪 ③ 设置失败重试机制。**适用场景**：投标文件自动化流水线。
- [ ] **方法2**：确保每个Agent专注单一任务，避免功能混杂。**实现方式**：为每个Agent编写专用提示词，限定其职责范围。**示例**："解析Agent"只负责从HTML提取字段，不负责筛选。
- [ ] **方法3**：建立输出自动流转机制，使用JSON作为中间格式。**技术要点**：每个Agent输出标准化JSON，下一个Agent直接读取。**好处**：减少人工干预。
```

**关键区别**：具体 vs 抽象，可操作 vs 空泛。

## 二、三个栏目的具体要求

### 1. 方法（提炼）—— **必须满足以下条件**
- **数量**：3-5条，不能少于3条
- **每条长度**：至少100字（包含具体步骤、适用场景、注意事项）
- **格式**：每条方法以`[ ] **方法X**：`开头，用**粗体**突出方法名
- **内容要求**：
  - **具体步骤**：如何操作（第一步、第二步...）
  - **适用场景**：这个方法可以用在什么工作中
  - **技术要点**：需要注意什么（如提示词设计、工具选择）
  - **示例**：最好能举例说明

### 2. 实践结果—— **必须引用原文数据**
- **要求**：必须是原文中提到的**具体数据、案例、实验结果**
- **不能**：抽象描述（如"效果很好"、"质量提升"）
- **必须**：具体事实（如"实验显示准确率提升30%"、"在3个项目中实际应用"）
- **格式**：每条结果以`[ ] **结果X**：`开头，引用原文内容

### 3. 达到的效果—— **必须量化**
- **要求**：说明这个方法带来的**量化成果**
- **不能**：模糊描述（如"提高效率"、"改善质量"）
- **必须**：具体数字（如"开发时间从3天缩短到1小时"、"人工审核工作量减少80%"）
- **格式**：每条效果以`[ ] **效果X**：`开头，包含具体数字

## 三、质量检查清单（麦龙自查）

提交卡片前，必须逐项检查：

### 方法栏检查
- [ ] 每条方法是否都有**具体步骤**？
- [ ] 是否说明了**适用场景**（能否用于我们的工作）？
- [ ] 是否有**技术要点**或**注意事项**？
- [ ] 每条方法是否达到**100字以上**？
- [ ] 方法总数是否在**3-5条**之间？

### 实践结果栏检查
- [ ] 每条结果是否**引用原文具体数据**？
- [ ] 是否有**实际案例**或**实验数据**？
- [ ] 是否避免了抽象描述？

### 效果栏检查
- [ ] 每条效果是否**量化**（有具体数字）？
- [ ] 是否说明了**实际价值**？
- [ ] 是否避免了模糊描述？

### 总体检查
- [ ] 这张卡片的内容能否**直接用于我们的工作**？
- [ ] 如果Boss问"这个方法怎么用"，能否回答清楚？
- [ ] 卡片是否达到"不需要再补充"的完整程度？
