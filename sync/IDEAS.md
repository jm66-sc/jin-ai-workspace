## 二、心跳节奏方案（麦龙建议，待 Cloud Code 确认）

### 什么是"轮回"（心跳）

> 所有 Agent 在同一个大框架下，以固定节奏同步"醒来"，
> 检查任务 → 读取信息 → 执行 → 汇报 → 继续等待。
> 就像心跳一样，到点就跳，一起跳，节奏不乱。

### 三档心跳节奏

| 档位 | 频率 | 适用场景 |
|------|------|---------|
| **快心跳** | 每 30 分钟 | Boss 有临时指令，或任务紧急 |
| **标准心跳** | 每 1 小时 | 日常标准节奏 |
| **慢心跳** | 每 3 小时 | 低优先级 / 后台监控 |

### 三个 Agent 的心跳配置（建议）

| Agent | 建议节奏 | 原因 |
|-------|---------|------|
| **麦龙** | 每小时（标准心跳） | 随时待命，微信响应快 |
| **金龙** | 每 30 分钟（快心跳） | 本地 Mac 可长时间运行，适合浏览器自动化 |
| **Cloud Code** | 每日 09:00 + 21:00 定时 + 按需 | 主要是深度讨论，不需要高频轮询 |

### 每个心跳周期内做什么

```
到了心跳时刻
    ↓
① 读取 sync/TASKS.md   → 看有什么新任务
② 读取 sync/IDEAS.md    → 看有什么新想法/讨论
③ 读取 memory/{date}.md  → 看今天有什么上下文
④ 执行属于我的任务
⑤ 把结果写入对应文件
⑥ 推送通知（Boss微信 / GitHub）
    ↓
等待下一个心跳
```

### 触发逻辑

```
麦龙心跳（每小时）   → 扫描 TASKS → 执行 → 写回
金龙心跳（每30分钟）  → 扫描 TASKS（金龙部分）→ 执行 → 写回
Cloud Code 定时       → 每日 09:00 / 21:00 → 深度讨论 → 更新 IDEAS + TASKS
Boss 临时指令         → 直接写入 TASKS → 下一次心跳时全员感知
```

### 这个系统的本质

> **整个系统在"大框架里自生长"**
> - 框架是死的（PROTOCOL.md）
> - 轮回节奏是固定的（心跳）
> - 信息在 GitHub 里流动（IDEAS / TASKS / sandbox）
> - 三个 Agent 按节奏读取、执行、汇报
> - Boss 在任何节点输入，整个系统就动起来
