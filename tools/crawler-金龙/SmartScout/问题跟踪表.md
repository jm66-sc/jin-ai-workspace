### #001: max_pages=0 边界处理缺陷
**状态**: 待处理
**发现时间**: 2026-03-15
**影响版本**: 所有版本
**优先级**: 中
**关联改动项**: max_pages参数传递修复

**问题描述**:
当API传入`max_pages=0`时，Producer接收参数正确，但运行时发生错误：
```
local variable 'page_num' referenced before assignment
```

**根本原因**:
`src/producer.py:312`:
```python
for page_num in range(1, self.max_pages + 1):
```
当`max_pages=0`时，`range(1, 1)`为空循环，变量`page_num`从未被赋值。

在`src/producer.py:329`:
```python
logger.info(f"总计爬取页数: {min(page_num, self.max_pages)}")
```
此处引用未定义的`page_num`变量，导致`UnboundLocalError`。

**当前影响**:
- `max_pages=0`表示"不抓取任何页面"的语义无法实现
- 任务启动后立即失败，影响任务状态跟踪
- 边界情况处理不完整

**修复建议**:
1. 在循环前初始化`page_num = 0`
2. 或添加边界检查：`if self.max_pages <= 0: return []`
3. 更新日志逻辑：使用已初始化的变量

**测试方案**:
1. 发送`max_pages=0`的生产请求
2. 验证Producer正确处理，不抓取页面
3. 验证任务正常完成（无错误）
4. 验证日志输出合理

**备注**:
- 此问题在max_pages参数传递修复项中发现
- 由于不影响主要功能，暂未在该项中修复
- 建议作为独立边界问题处理
