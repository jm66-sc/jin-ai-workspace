## 2. 改动前后的统计口径对照

### 改动前 (问题状态)
| 字段 | 含义 | 问题 |
|------|------|------|
| `skipped_no_rule` | "无规则匹配"跳过 | ✗ 字段名误导，实际业务中未匹配规则默认通过，不是跳过 |
| `passed_whitelist` | "通过的任务（混合两种通过原因）" | ✗ 含义模糊，包含两种不同情况：真正匹配白名单和默认通过 |
| 无 | 统计默认通过 | ✗ 未匹配规则的任务没有专门统计字段 |

### 改动后 (修复状态)
| 字段 | 含义 | 状态 |
|------|------|------|
| `skipped_no_rule` | "其他跳过" (保留字段) | ✓ 保留但注明"目前永远为0"，用于未来扩展 |
| `passed_whitelist` | "所有通过的任务" | ✓ 保持向后兼容，值为`passed_whitelist_matched + passed_default` |
| `passed_whitelist_matched` | "真正匹配白名单的任务" | ✓ 新增字段，准确统计匹配白名单的任务 |
| `passed_default` | "未匹配规则但默认通过的任务" | ✓ 新增字段，准确统计默认通过的任务 |

**统计关系公式**:
- `passed_whitelist = passed_whitelist_matched + passed_default`
- `skipped_no_rule = 0` (保留字段)
- `total_skipped = skipped_blacklist + skipped_no_rule = skipped_blacklist`

| 标题 | 业务行为 | 旧统计字段 | 新统计字段 |
|------|----------|------------|------------|
| 消防设备采购 | 通过 (匹配白名单) | `passed_whitelist+1` | `passed_whitelist_matched+1` |
| 食堂配送服务 | 跳过 (命中黑名单) | `skipped_blacklist+1` | `skipped_blacklist+1` |
| 办公用品采购 | 通过 (默认通过) | `passed_whitelist+1` (错误分类) | `passed_default+1` (正确分类) |

**验证结论**: 新统计字段准确反映了实际业务行为，解决了"未匹配规则默认通过"与统计显示的矛盾。
