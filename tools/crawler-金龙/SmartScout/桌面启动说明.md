# SmartScout 桌面启动器使用说明

## 📦 已完成改进

### 1. 修复端口配置
- 修复了 `run_all.sh` 脚本中的端口不匹配问题
- 前端端口从 `3000` 改为 `3001`（与 `vite.config.js` 配置一致）

### 2. 自动打开浏览器
- 系统启动完成后自动打开浏览器访问 `http://localhost:3001`
- 无需手动输入地址

### 3. 桌面启动器脚本
- 创建了 `SmartScout-Launcher.sh` 一键启动脚本
- 支持检测服务是否已在运行，避免重复启动

## 🚀 使用方法

### 方案一：使用桌面启动器（推荐）

**步骤 1：复制到桌面**
```bash
# 复制启动器到桌面
cp "/Users/jin/Desktop/爬虫工具/SmartScout/SmartScout-Launcher.sh" ~/Desktop/
```

**步骤 2：设置执行权限**
```bash
# 如果从终端运行，需要设置权限
chmod +x ~/Desktop/SmartScout-Launcher.sh
```

**步骤 3：运行方式**

**方式 A：从终端运行**
```bash
# 打开终端
cd ~/Desktop
./SmartScout-Launcher.sh
```

**方式 B：双击运行（macOS）**
1. 右键点击 `SmartScout-Launcher.sh`
2. 选择"打开方式" → "其他..."
3. 选择"终端"（或"iTerm"）
4. 勾选"始终以此方式打开"
5. 点击"打开"

**方式 C：创建别名（方便后续使用）**
```bash
# 添加到 ~/.zshrc 或 ~/.bashrc
alias smartscout='cd "/Users/jin/Desktop/爬虫工具/SmartScout" && ./run_all.sh'
```

### 方案二：直接运行主脚本

```bash
# 打开终端，进入项目目录
cd "/Users/jin/Desktop/爬虫工具/SmartScout"

# 运行一键启动脚本
./run_all.sh
```

## ⚠️ 注意事项

1. **首次运行**：需要安装依赖
   - 前端：自动运行 `npm install`
   - 后端：自动运行 `pip install -r requirements_api.txt`

2. **端口占用**：如果端口被占用，启动会失败
   - 3001端口：前端开发服务器
   - 8000端口：后端FastAPI服务

3. **停止系统**：按 `Ctrl+C` 停止所有服务

4. **查看日志**：日志文件保存在 `logs/` 目录
   - `backend.log`：后端日志
   - `frontend.log`：前端日志

## 🔧 故障排除

### 问题：双击脚本无法运行
**解决方案**：
```bash
# 方法1：从终端运行
open -a Terminal ~/Desktop/SmartScout-Launcher.sh

# 方法2：创建快捷方式
echo 'cd "/Users/jin/Desktop/爬虫工具/SmartScout" && ./run_all.sh' > ~/Desktop/start-smartscout.command
chmod +x ~/Desktop/start-smartscout.command
```

### 问题：端口已被占用
**解决方案**：
```bash
# 查看占用3001端口的进程
lsof -i :3001

# 查看占用8000端口的进程
lsof -i :8000

# 杀死占用进程（谨慎操作）
kill -9 <PID>
```

### 问题：依赖安装失败
**解决方案**：
```bash
# 手动安装前端依赖
cd frontend
npm install --legacy-peer-deps

# 手动安装后端依赖
pip install -r requirements_api.txt
```
