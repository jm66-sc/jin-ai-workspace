### 规则 1：入库去重

**问题：** 同一 URL 被多次写入数据库，导致重复记录

**规则：**
> 所有入库操作必须有**唯一键去重**（URL/doc_id），使用 `INSERT OR IGNORE` 或先 `SELECT` 检查存在性

**实现方式：**
```python
# 方式一：INSERT OR IGNORE
cursor.execute("INSERT OR IGNORE INTO results (url, ...) VALUES (?, ...)", (url, ...))

# 方式二：先检查
cursor.execute("SELECT 1 FROM results WHERE url = ?", (url,))
if not cursor.fetchone():
    cursor.execute("INSERT INTO results ...")
```

**后续动作：** 写入 SmartScout 设计文档的"数据入库"章节

---

### 规则 2：快速模式优先

**问题：** DeepSeek API 提取字段需要 3-5 秒/条，太慢

**规则：**
> 默认使用**快速模式**（只抓标题+URL入库），深度模式（调用API提取字段）作为可选增强，由用户按需触发

**实现方式：**
- 默认：`python crawl.py --mode=quick`
- 深度：`python crawl.py --mode=deep --select=项目ID列表`

**后续动作：** 写入 SmartScout 设计文档的"运行模式"章节

## 二、候选规则

**标注：** 【候选规则】
**状态：** 需要 2-3 轮验证后才能确认
**理由：** 问题存在但样本不足，或解决方案需要更多验证

---

### 候选规则 1：黑白名单精准化

**问题：** "机电设备"关键词太宽泛，抓到很多无关内容（物业管理、餐饮服务等）

**候选规则：**
> - 黑名单要**排除招标公司名字干扰**（如"xxx招标有限公司"）
> - 白名单用**具体业务词**（如"电气安装"、"弱电系统维护"）而非泛词（如"机电"）

**需要验证：**
- 换 2-3 组关键词测试
- 验证过滤准确率是否稳定提升到 > 90%

**后续动作：** 验证通过后写入设计文档的"规则配置"章节

---

### 候选规则 2：字段完整性校验

**问题：** 很多入库记录只有标题，没有采购单位、预算金额等关键字段

**候选规则：**
> 字段提取后必须有**完整性校验**，少于 N 个字段标记为"待补全"，进入重试队列

**需要验证：**
- 统计 DeepSeek API 返回空/不全的频率
- 确定合理的 N 值（关键字段数量阈值）
- 确认重试机制是否有效

**后续动作：** 验证通过后写入设计文档的"字段提取"章节

---

### 候选规则 3：Consumer 自动启动

**问题：** Consumer 不会自动启动，需要手动触发

**候选规则：**
> Consumer 应在 Producer 完成后**自动触发**，或作为后台常驻进程运行

**需要验证：**
- 再跑 2-3 次完整流程
- 确认当前手动触发是否是设计缺陷还是有意为之
- 测试自动触发的稳定性

**后续动作：** 验证通过后写入设计文档的"Pipeline 流程"章节
