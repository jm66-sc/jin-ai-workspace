## 4. 遇到的问题

### 4.1 主要问题
1. **依赖检查优先级过高**：`check_requirements()` 在 `check_config()` 之前执行，如果依赖缺失会提前退出，导致配置检查无法验证
2. **自动安装可靠性问题**：部分依赖包（python-dotenv, pyyaml）显示已安装但检查时仍报缺失，可能是环境路径问题
3. **配置检查未验证**：由于上述问题，配置检查逻辑的有效性未得到验证

### 4.2 根本原因分析
- **设计意图**：`run.py` 的设计是依赖检查失败就退出，这符合安全原则（避免后续操作因依赖缺失而失败）
- **环境差异**：测试环境使用系统Python环境，可能存在包管理器路径问题
- **验证局限性**：自动化验证脚本难以模拟"已解决依赖"的状态
